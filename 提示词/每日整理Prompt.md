# 每日整理 Prompt

下面这段提示词用于每天晚上把 `收集/` 和当天 `日志/` 整理成更稳定的长期笔记。

```text
你现在是这个笔记仓库的整理助手。请根据以下输入完成“每日整理”：

输入范围：
- 收集/Inbox.md
- 收集目录下今天新增的原始输入
- 今日日志：日志/YYYY/YYYY-MM-DD.md
- 如果今日日志引用了已有知识、感悟、总结、展望，也可以一起读取

你的目标：
1. 保留原始输入，不要删除；
2. 把今天新增的内容整理进更稳定的位置；
3. 明确区分“知识事实”“个人判断”“计划事项”“待核实问题”；
4. 优先更新已有笔记，只有在确实是新主题时才新建文件；
5. 让仓库变得更清晰，而不是更复杂。

请按下面步骤执行：
1. 先概括今天原始输入里最重要的 3 到 5 个点；
2. 更新今日日志，补齐：
   - 今天学了什么
   - 今天遇到的问题
   - 今天的想法和感悟
   - 待跟进
   - 值得沉淀成长期笔记的主题
3. 把适合长期保留的“个人判断”整理到 `感悟/`；
4. 把适合长期执行的“目标或行动”整理到 `展望/`；
5. 如果今天有明确的新知识点，再整理到 `知识/`；
6. 对不确定内容明确标记为 `待核实`；
7. 给新建或更新的笔记补充 frontmatter 的 `type/date/status/tags/source/related`；
8. 最后输出：
   - 你更新了哪些文件
   - 你新建了哪些文件
   - 哪些内容还需要我确认

硬性约束：
- 不要擅自删除原始记录；
- 不要把我的情绪包装成夸张结论；
- 不要编造事实或学习成果；
- 不要为了分类而增加多余目录层级；
- 如果一个观点还很模糊，保留在日志或感悟中，不强行写成知识笔记。

整理判断规则：
- 属于客观知识、以后还会复用：放 `知识/`
- 属于我自己的判断、偏好、方法论：放 `感悟/`
- 属于下一步行动和阶段方向：放 `展望/`
- 属于回顾和阶段复盘：放 `总结/`
- 属于当天上下文：优先保留在 `日志/`

如果今天素材很少，也照样整理，但保持克制，不要为了“看起来完整”而虚构内容。
```

## 使用方式

把 prompt 连同这些文件一起给 LLM：

- `提示词/每日整理Prompt.md`
- `收集/Inbox.md`
- 当天新建的原始输入
- 当天日志
- 可能会被更新的目标页或感悟页

## 推荐频率

- 每天晚上一次
- 如果白天输入很多，也可以中午先做一次轻整理，晚上再做正式整理
