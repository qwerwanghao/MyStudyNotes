# 月总结 Prompt

这份 prompt 只负责“单次执行”。

长期规则、目录约束、frontmatter 规范、允许与禁止事项，统一以 [LLM维护手册.md](../LLM维护手册.md) 为准。

```text
你现在要执行一次“月总结”。

在开始前，先读取并遵循：
- LLM维护手册.md

本次输入范围：
- 本月的日志
- 本月的周回顾
- 本月新增或更新过的感悟、知识、展望笔记
- 如果已有相关月总结，也可以一起读取

本次任务目标：
1. 从本月输入中提炼出真正有积累的部分；
2. 生成一份诚实、克制、可复核的月总结；
3. 把未完成事项转成下个月值得继续追踪的方向；
4. 不夸大学习成果，不美化情绪，不删除原始材料。

请按下面步骤执行：
1. 概括本月最重要的 3 到 5 个主题；
2. 归纳：
   - 这个月发生了什么
   - 真正有积累的部分
   - 哪些地方仍在反复卡住
   - 哪些认知或判断发生了变化
   - 下个月最值得继续推进的方向
3. 生成或更新本月总结到 `总结/`
4. 如有必要，把未完成事项同步到 `展望/`
5. 列出仍然需要我确认的内容

最后输出：
- 你更新了哪些文件
- 你新建了哪些文件
- 哪些内容还需要我确认
```

## 使用方式

把这份 prompt 连同这些文件一起交给 LLM：

- `提示词/月总结Prompt.md`
- `LLM维护手册.md`
- 本月日志
- 本月周回顾
- 相关的感悟 / 知识 / 展望
- 可能会被更新的月总结
