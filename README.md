# MyStudyNotes

> 一个由我负责记录、由 LLM 协助整理和维护的长期知识库。

## 这次重构解决什么问题

这个仓库不再要求我先想清楚分类、命名、归档，再开始写。

新的原则只有三条：

1. 先记录，再整理。
2. 原始输入和长期沉淀分层存放。
3. LLM 负责提炼、归档、串联和维护索引，不伪造我的观点。

## 仓库结构

- [收集/Inbox.md](收集/Inbox.md)：最低摩擦入口，任何碎片都先丢进来。
- [日志](日志)：每天一页，把学习、问题、任务、感悟先落地。
- [知识](知识)：长期知识笔记，适合沉淀成 evergreen notes。
  入口见 [知识/README.md](知识/README.md)，知识目录统一为“大类 / 主题 / 分组 / 资源”。
- [感悟](感悟)：更偏个人判断、认知变化、人生体会。
- [总结](总结)：周、月、年、阶段复盘。
- [展望](展望)：阶段目标、年度方向、下一步重点。
- [索引/工作台.md](索引/工作台.md)：导航页，集中放当前入口和高频动作。
- [模板](模板)：统一模板，降低开写成本。
- [LLM维护手册.md](LLM维护手册.md)：告诉 LLM 应该怎么维护这个仓库。

## 每天怎么用

### 1. 白天只做记录

优先把内容写到下面两个地方之一：

- 临时碎片： [收集/Inbox.md](收集/Inbox.md)
- 当天正式记录： [日志/2026/2026-04-12.md](日志/2026/2026-04-12.md)

记录时不要强迫自己整理，只要把这些写下来：

- 今天学了什么
- 遇到了什么问题
- 有什么还没想清楚
- 产生了什么判断和感悟
- 哪些内容值得以后沉淀成长期笔记

### 2. 晚上让 LLM 整理

把 `Inbox` 和当天日志交给 LLM，让它做这些事：

- 抽取新增知识点，必要时写入 `知识/`
- 抽取明确的个人判断，必要时写入 `感悟/`
- 归并重复内容
- 给笔记补链接、标签、状态
- 把待办问题单独列出来

### 3. 每周做一次回顾

让 LLM 基于最近 7 天的日志生成：

- 学到了什么
- 哪些点还模糊
- 哪些内容值得升级成长期笔记
- 下周最值得投入的方向

## 推荐维护节奏

- 每天：写 `Inbox` 或当天日志
- 每周：生成周回顾，整理知识和感悟
- 每月：从周回顾和日志生成月总结
- 每年：从月总结和日志生成年度总结与下一年展望

## 维护规则

- 原始记录优先保留，不直接删除
- 知识事实和个人观点分开写
- LLM 不替我发明结论，不确定的内容标记为 `待核实`
- 一篇长期笔记尽量只讲一个主题
- 导航依赖索引页，而不是无限加深目录层级
