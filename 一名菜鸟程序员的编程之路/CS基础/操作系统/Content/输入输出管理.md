# 输入输出管理

## I/O设备管理

### I/O设备分类
1. **按传输速率分类**
   - 低速设备
   - 中速设备
   - 高速设备

2. **按信息交换单位分类**
   - 块设备
   - 字符设备
   - 网络设备

3. **按设备共享属性分类**
   - 独占设备
   - 共享设备
   - 虚拟设备

### I/O控制方式
1. **程序直接控制**
   - CPU轮询
   - 等待设备就绪
   - CPU利用率低
   - 适用于简单设备

2. **中断驱动方式**
   - 设备完成触发中断
   - CPU可执行其他任务
   - 效率较高
   - 适用于字符设备

3. **DMA方式**
   - 直接存储器访问
   - 数据块传输
   - CPU介入少
   - 适用于块设备

4. **通道控制方式**
   - 专用处理器
   - 执行通道程序
   - CPU完全独立
   - 适用于复杂I/O系统

## 缓冲管理

### 缓冲区类型
1. **单缓冲**
   - 一个缓冲区
   - 轮流使用
   - 实现简单
   - 效率较低

2. **双缓冲**
   - 两个缓冲区
   - 交替使用
   - 提高效率
   - 空间开销大

3. **循环缓冲**
   - 多个缓冲区
   - 环形队列
   - 生产消费模型
   - 灵活管理

### 缓冲区管理策略
1. **分配策略**
   - 固定分配
   - 动态分配
   - 池化管理
   - 按需调整

2. **回收策略**
   - 立即回收
   - 延迟回收
   - 阈值触发
   - 定期清理

3. **替换策略**
   - LRU算法
   - FIFO算法
   - 时钟算法
   - 优先级算法

## 设备调度

### 磁盘调度算法
1. **先来先服务(FCFS)**
   - 按请求顺序
   - 公平简单
   - 性能一般
   - 寻道时间可能长

2. **最短寻道时间优先(SSTF)**
   - 选择最近的请求
   - 减少寻道时间
   - 可能产生饥饿
   - 性能较好

3. **扫描算法(SCAN)**
   - 电梯算法
   - 单向扫描
   - 防止饥饿
   - 效率较高

4. **循环扫描(C-SCAN)**
   - 单向服务
   - 循环往复
   - 更公平
   - 适用于负载重的系统

### 打印机调度
1. **先来先服务**
   - 按作业到达顺序
   - 实现简单
   - 对大作业不利

2. **最短作业优先**
   - 最少页数优先
   - 提高吞吐量
   - 可能延迟大作业

3. **优先级调度**
   - 重要性排序
   - 紧急程度
   - 灵活性好
   - 需要优先级管理
