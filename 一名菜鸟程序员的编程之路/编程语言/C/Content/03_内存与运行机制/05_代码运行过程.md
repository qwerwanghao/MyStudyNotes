# 学习笔记：代码运行过程

## 一、运行模型

- 运行模型描述了程序执行时的各个阶段，包括代码段、数据段、堆、栈等。

![alt text](../Media/运行模型.png)

---

## 二、代码段

1. **编译过程**:
   - 编译器将 C 语言代码编译成二进制指令代码。

2. **加载代码段**:
   - 程序运行时，将二进制代码加载到代码段。

3. **代码段特性**:
   - 代码段是只读的，不可被修改，以提高安全性。

4. **指令指针**:
   - CPU 有一个指令指针，指向当前执行的指令。

5. **指令执行**:
   - CPU 执行完一条指令后，会移动指令指针到下一个位置。

6. **条件跳转**:
   - 支持条件跳转，根据执行结果选择下一条指令。

7. **循环跳转**:
   - 支持循环跳转，以执行循环体。

8. **函数调用跳转**:
   - 支持函数调用跳转、函数返回跳转和参数传递。

---

## 三、数据段

1. **数据段分配**:
   - 每个进程在运行时都会分配一个数据段。

2. **存储全局变量**:
   - 数据段存放程序中的所有全局变量，并为其分配内存。

3. **内存限制**:
   - 数据段内存一旦分配好，不能增加或减少。

---

## 四、栈

1. **栈的分配**:
   - 操作系统为每个进程分配一个栈。

2. **栈的大小**:
   - 栈的大小有限，不宜在栈上分配大规模的内存块。

3. **栈操作**:
   - 栈底指针向下移动分配内存，向上移动回收内存。

4. **局部变量与参数**:
   - 局部变量和函数参数的内存都分配在栈上。

---

## 五、关键点

### 1. 分配局部变量与参数
- 当函数被调用时，为局部变量和参数分配内存，通常在栈上进行。
- 栈是后进先出（LIFO）的数据结构，栈底指针（ESP或RSP寄存器）指向当前栈的顶部。
- 栈底指针向下移动，为局部变量和参数分配内存。

### 2. 程序运行时的栈大小有限
- 栈的大小有限，避免定义大块内存的局部变量以防止栈溢出。
- 若需要大块内存，考虑使用堆内存（通过 `malloc` 或 `new` 等动态分配函数）。

### 3. 函数调用
- 在调用函数时进行一系列操作来准备和管理函数调用的上下文：
  - **分配参数内存**: 从右到左分配参数内存。
  - **初始化参数**: 将实际参数值存储到分配的内存中。
  - **保存返回地址**: 将下一条指令地址保存到栈中，以便返回时继续执行。
  - **保存当前栈底指针**: 保存当前栈底指针位置以便返回时恢复。
  - **跳转到函数地址**: 设置指令指针为被调用函数的地址，开始执行。

### 4. 函数返回
- 当函数执行完毕，需要返回到调用函数的地方，并恢复调用前的状态：
  1. **回收局部变量**: 释放函数内部局部变量占用的栈空间，恢复栈底指针。
  2. **恢复指令指针**: 将指令指针恢复到保存的返回地址，继续执行。
  3. **保存返回值**: 将函数返回值存储到特定寄存器（如 EAX 或 RAX）中，以便调用者使用。

## 六、举例演示

### 示例代码
```c
int add(int a, int b) {
    int result = a + b;
    return result;
}

int main() {
    int x = 5;
    int y = 10;
    int sum = add(x, y);
    return 0;
}
```

### 运行过程分析

1. **分配局部变量与参数**:
   - `main` 函数开始执行，分配局部变量 `x` 和 `y` 的内存。
   - `x` 和 `y` 分别被初始化为 5 和 10。

2. **函数调用**:
   - `main` 函数调用 `add(x, y)`。
   - 从右到左分配参数的内存：
     - 首先为 `b` 分配内存，然后为 `a` 分配内存。
     - 将 `y` 的值（10）存储到 `b` 的内存位置，将 `x` 的值（5）存储到 `a` 的内存位置。
   - 保存 `main` 函数中调用 `add` 函数的下一条指令地址到栈中。
   - 保存当前栈底指针的位置。
   - 跳转到 `add` 函数的地址，开始执行 `add` 函数。

3. **函数执行**:
   - `add` 函数开始执行，分配局部变量 `result` 的内存。
   - 计算 `a + b` 的值（5 + 10 = 15），并将结果存储到 `result` 中。
   - 返回 `result` 的值（15），并将其存储到 `EAX` 寄存器中。

4. **函数返回**:
   - 回收 `add` 函数的局部变量 `result` 的内存，将栈底指针恢复到调用前的位置。
   - 恢复指令指针到 `main` 函数中调用 `add` 函数的下一条指令。
   - `main` 函数继续执行，将 `EAX` 寄存器中的返回值（15）存储到 `sum` 中。

### 汇编过程示例

```assembly
main:
    ; 分配局部变量 x 和 y
    mov dword [esp-4], 5    ; x = 5
    mov dword [esp-8], 10   ; y = 10

    ; 调用 add 函数
    push dword [esp-8]      ; push y
    push dword [esp-4]      ; push x
    call add                ; 调用 add 函数

    ; 函数返回后，继续执行下一条指令
    mov [esp-12], eax       ; sum = 返回值 (eax)
    mov eax, 0              ; return 0
    ret                     ; 返回

add:
    ; 函数体
    push ebp                ; 保存调用者的栈底指针
    mov ebp, esp            ; 设置当前栈底指针
    sub esp, 4              ; 分配局部变量 result

    ; 计算 a + b
    mov eax, [ebp+8]        ; eax = a
    add eax, [ebp+12]       ; eax += b
    mov [ebp-4], eax        ; result = eax

    ; 返回 result
    mov eax, [ebp-4]        ; eax = result
    mov esp, ebp            ; 恢复栈底指针
    pop ebp                 ; 恢复调用者的栈底指针
    ret                     ; 返回
```
