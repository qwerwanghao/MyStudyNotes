# C语言流程控制

## 条件语句

### if-else语句
```c
if (condition1) {
    // 代码块1
} else if (condition2) {
    // 代码块2
} else {
    // 默认代码块
}

// 单行if的大括号使用
if (condition)
    statement;  // 不推荐省略大括号
```

### switch语句
```c
switch (expression) {
    case constant1:
        // 代码块1
        break;  // 不要忘记break
    case constant2:
        // 代码块2
        break;
    default:
        // 默认代码块
}
```

## 循环语句

### while循环
```c
// 基本while循环
while (condition) {
    // 循环体
}

// do-while循环
do {
    // 至少执行一次
} while (condition);
```

### for循环
```c
// 标准for循环
for (initialization; condition; increment) {
    // 循环体
}

// 多重初始化/增量
for (i = 0, j = n-1; i < j; i++, j--) {
    // 回文字符串检查
}
```

## 跳转语句

### break和continue
```c
// break用法
while (1) {
    if (error_condition) {
        break;  // 跳出循环
    }
}

// continue用法
for (i = 0; i < n; i++) {
    if (skip_condition) {
        continue;  // 跳过本次循环
    }
    // 处理代码
}
```

### goto语句（谨慎使用）
```c
// 错误处理场景
if (error1)
    goto cleanup;
if (error2)
    goto cleanup;

// 正常处理

cleanup:
    // 释放资源
    return error_code;
```

## 流程控制最佳实践

### 1. 深度控制
```c
// 减少嵌套深度
if (error_condition) {
    return error_code;  // 早返回
}
// 正常处理逻辑
```

### 2. 循环控制
```c
// 循环不变量
while (left < right) {
    // 保持循环不变量
    left++;
    right--;
}

// 避免死循环
while (condition) {
    if (timeout_condition) {
        break;  // 确保有退出机制
    }
}
```

### 3. 开关优化
```c
// 使用查找表替代多重if-else
static const int lookup[] = {
    [KEY1] = VALUE1,
    [KEY2] = VALUE2
};
return lookup[key];
```

## 常见错误处理模式

### 1. 错误码返回
```c
int function() {
    if (error_condition) {
        return ERROR_CODE;
    }
    return SUCCESS;
}
```

### 2. 错误标志设置
```c
bool success = true;
if (!operation1()) {
    success = false;
    goto cleanup;
}
if (!operation2()) {
    success = false;
    goto cleanup;
}

cleanup:
    // 清理资源
    return success;
```

## 性能优化提示

### 1. 循环优化
```c
// 将不变计算移出循环
size_t len = strlen(str);  // 计算一次
for (i = 0; i < len; i++) {
    // 使用len而不是反复调用strlen
}
```

### 2. 分支预测
```c
// 将最可能的情况放在前面
if (likely_condition) {   // 更可能发生的情况
    // 常见情况处理
} else {
    // 罕见情况处理
}
```

> 提示：良好的流程控制可以提高代码的可读性和维护性。
