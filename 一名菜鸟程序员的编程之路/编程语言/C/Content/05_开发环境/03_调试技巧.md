# C语言调试技巧总结

## 常见错误类型

### 1. 编译时错误
- 语法错误
- 未声明的标识符
- 类型不匹配
- 缺少头文件

### 2. 运行时错误
- 段错误(Segmentation Fault)
- 栈溢出
- 内存泄漏
- 除零错误

## 调试工具使用

### GDB基本命令
```bash
gdb ./程序名
(gdb) break main     # 在main函数设置断点
(gdb) run           # 运行程序
(gdb) next         # 单步执行
(gdb) print 变量名  # 打印变量值
(gdb) bt           # 查看调用栈
```

### 内存检测
```bash
# Valgrind使用示例
valgrind --leak-check=full ./程序名
```

## 打印调试技巧

### 1. 格式化打印
```c
// 使用__FILE__和__LINE__宏
#define DEBUG_PRINT(fmt, ...) \
    printf("[%s:%d] " fmt "\n", __FILE__, __LINE__, ##__VA_ARGS__)

// 使用示例
DEBUG_PRINT("变量x的值: %d", x);
```

### 2. 条件编译
```c
#ifdef DEBUG
    printf("调试信息\n");
#endif

// 编译时使用
// gcc -DDEBUG 程序.c
```

## 最佳实践

### 1. 错误处理
```c
// 推荐写法
if (ptr == NULL) {
    perror("内存分配失败");
    return -1;
}
```

### 2. 断言使用
```c
#include <assert.h>
// 用于检查不应该发生的情况
assert(ptr != NULL);
assert(size > 0);
```

### 3. 内存检查
```c
void *p = malloc(size);
if (!p) {
    // 错误处理
    return NULL;
}
// 使用完后
free(p);
p = NULL;  // 避免悬挂指针
```

## 常见问题诊断

### 1. 内存相关
- 使用未初始化的指针
- 缓冲区溢出
- 重复释放
- 内存泄漏

### 2. 指针相关
- NULL指针解引用
- 野指针访问
- 指针越界
- 指针类型不匹配

## 调试工具比较

| 工具 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| printf | 简单直接 | 需要修改代码 | 简单问题 |
| GDB | 功能强大 | 学习曲线陡 | 复杂问题 |
| Valgrind | 内存检测全面 | 运行慢 | 内存问题 |
| AddressSanitizer | 速度快 | 需要重编译 | 开发期 |
