# 学习笔记：C 语言加载过程

在 C 语言中，加载过程涉及将可执行文件从外存加载到内存，并使程序能够运行。以下是 Linux 系统中进程运行时的内存映像和加载过程的详细说明。

## 内存映像

在 Linux 中，进程运行时的内存映像通常包括以下几个部分：

1. **代码段**: 存放程序的机器指令。
2. **数据段**: 存放已初始化的全局变量和静态变量。
3. **BSS 段**: 存放未初始化的全局变量和静态变量。
4. **堆**: 动态分配的内存区域。
5. **栈**: 存放函数调用时的局部变量和函数调用信息。


## 加载过程

![alt text](../Media/内存映像.png)

加载过程如下：

![alt text](../Media/加载过程.png)

1. **创建内存映像**:
   - 加载器（Loader）首先创建进程的内存映像。内存映像包括代码段、数据段、BSS 段、堆和栈等。
   - 加载器根据段头部表（Program Header Table），将目标文件的内容拷贝到相应的内存区域。目标文件中定义的代码和数据会被加载到内存中的数据段和代码段。

2. **执行启动代码**:
   - 加载器完成内存映像的创建后，跳转到程序的入口点（通常是符号 `_start` 的地址）。
   - 启动代码（Startup Code）包括以下步骤：
     1. **初始化堆栈**: 设置程序的堆栈指针。
     2. **初始化全局数据**: 初始化数据段和 BSS 段中的变量。
     3. **调用 C 运行时库（CRT）初始化函数**: 这些函数会进行进一步的初始化，比如处理全局构造函数和析构函数等。
     4. **调用 `main` 函数**: 程序的实际入口点，用户定义的 `main` 函数会在此被调用。
   
   启动代码的调用顺序确保程序在执行用户代码之前完成所有必要的初始化步骤。

## 总结

- **加载器**负责将可执行文件从外存加载到内存，并进行必要的初始化。
- **内存映像**包含程序运行所需的各个内存区域。
- **启动代码**完成程序的初始化，并最终调用用户定义的 `main` 函数，开始程序的执行。

了解加载过程有助于理解程序在运行时的内存管理及其生命周期管理。
