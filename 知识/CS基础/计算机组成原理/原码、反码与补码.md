# 学习笔记：原码、反码与补码

---
## 一、基本概念

| **概念** | **说明** |
|----------|----------|
| **原码** | 用于表示有符号整数的二进制编码方式，最高位为符号位（0表示正数，1表示负数），其余位表示数值大小。 |
| **反码** | 原码的基础上，负数的反码是其原码除符号位外各位取反，正数的反码与原码相同。 |
| **补码** | 反码的基础上，负数的补码是在反码的基础上加1，正数的补码与原码相同。 |

---
## 二、编码方式详解

### 1. 原码（Sign-Magnitude）

- **定义**：最高位为符号位，0表示正数，1表示负数，其余位表示数值大小。
- **优点**：
  - 表示直观，易于理解。
- **缺点**：
  - 存在两个表示零（+0和-0）。
  - 计算复杂，需要分开处理符号和数值部分。

**示例：8位表示**

| **数值** | **原码**          |
|----------|-------------------|
| +5       | `0 0000101`       |
| -5       | `1 0000101`       |

### 2. 反码（One's Complement）

- **定义**：正数的反码与原码相同，负数的反码是其原码除符号位外各位取反。
- **优点**：
  - 与原码相比，负数的表示更为统一。
- **缺点**：
  - 仍存在两个表示零（+0和-0）。
  - 加法运算时需要处理“进位到符号位”的情况。

**示例：8位表示**

| **数值** | **原码**    | **反码**       |
|----------|-------------|----------------|
| +5       | `0 0000101` | `0 0000101`     |
| -5       | `1 0000101` | `1 1111010`     |

### 3. 补码（Two's Complement）

- **定义**：正数的补码与原码相同，负数的补码是在反码的基础上加1。
- **优点**：
  - 只有一个零，简化了计算。
  - 加减运算可以统一使用加法器，硬件实现简单。
- **缺点**：
  - 对初学者理解起来稍复杂。

**示例：8位表示**

| **数值** | **原码**    | **反码**       | **补码**        |
|----------|-------------|----------------|-----------------|
| +5       | `0 0000101` | `0 0000101`     | `0 0000101`      |
| -5       | `1 0000101` | `1 1111010`     | `1 1111011`      |

---
## 三、补码的详细解释

### 1. 补码的计算方法

| **步骤** | **说明** |
|----------|----------|
| **步骤1** | 取正数的原码。 |
| **步骤2** | 取反（符号位不变，其余位取反）。 |
| **步骤3** | 在取反结果上加1，得到补码。 |

**示例：计算-5的补码（8位）**

1. **原码**：`1 0000101`
2. **反码**：`1 1111010`
3. **补码**：`1 1111010` + `1` = `1 1111011`

### 2. 补码的优点

- **唯一零**：避免了原码和反码的双零问题。
- **简化运算**：加减运算统一为加法，硬件实现更高效。
- **溢出检测**：通过符号位的变化可以检测溢出。

### 3. 补码运算示例

**计算 5 + (-5)**

| **数值** | **补码**        |
|----------|-----------------|
| +5       | `00000101`      |
| -5       | `11111011`      |
| **加法** | `00000101` + `11111011` = `1 00000000`（忽略进位） = `00000000` |

**结果**：0

## 四、补码与原码、反码的对比

| **特性**       | **原码**                          | **反码**                            | **补码**                            |
|----------------|-----------------------------------|-------------------------------------|-------------------------------------|
| **符号位**     | 单独表示，0为正，1为负             | 单独表示，0为正，1为负               | 单独表示，0为正，1为负               |
| **零的表示**   | +0 和 -0                           | +0 和 -0                             | 只有 +0                              |
| **加减运算**   | 需分开处理符号和数值                 | 需要处理“进位到符号位”                | 统一使用加法器，简化运算               |
| **硬件实现**   | 复杂                               | 较复杂                               | 简单，高效                            |
| **溢出检测**   | 需额外逻辑                         | 需额外逻辑                           | 通过符号位变化自动检测                |

## 五、补充知识点

### 1. 补码的范围

| **位数** | **数值范围**                         |
|----------|--------------------------------------|
| 8位      | -128 到 +127                          |
| 16位     | -32,768 到 +32,767                    |
| 32位     | -2,147,483,648 到 +2,147,483,647      |

### 2. 补码的应用

- **计算机内部表示**：所有有符号整数在计算机中均使用补码表示。
- **算术运算**：加减乘除等算术运算统一使用补码，简化了硬件设计。
- **逻辑运算**：位运算如与、或、非等也基于补码进行。

### 3. 补码与溢出

- **溢出检测**：当两个正数相加得到负数，或两个负数相加得到正数时，发生溢出。
- **示例**：在8位补码中，`+127 + +1` 应该是 `+128`，但8位补码无法表示，结果会溢出为 `-128`。

### 4. C语言中的补码

- **`int8_t` 类型**：用于表示8位有符号整数，默认使用补码表示。
- **打印二进制**：通过位运算打印数值的二进制表示。

**示例代码**：

```c
#include <stdio.h>
#include <stdint.h>

int main() {
    // 正数示例
    int8_t positiveNum = 5; // 补码：00000101
    printf("正数示例:\n");
    printf("十进制: %d, 二进制: ", positiveNum);
    for (int i = 7; i >= 0; i--) {
        printf("%d", (positiveNum >> i) & 1);
    }
    printf("\n");

    // 负数示例
    int8_t negativeNum = -5; // 补码：11111011
    printf("\n负数示例:\n");
    printf("十进制: %d, 二进制: ", negativeNum);
    for (int i = 7; i >= 0; i--) {
        printf("%d", (negativeNum >> i) & 1);
    }
    printf("\n");

    // 补码运算示例
    int8_t sum = positiveNum + negativeNum; // 0
    printf("\n补码运算示例:\n");
    printf(" %d + (%d) = %d\n", positiveNum, negativeNum, sum);

    return 0;
}
```

## 六、是否将原码、反码与补码合并整理

**必要性**：

- **关联性强**：原码、反码与补码是理解有符号数表示及计算机算术运算的基础，彼此紧密相关。
- **逻辑连贯**：学习补码通常需要先了解原码和反码，以便理解补码的产生及其优势。

**建议**：将原码、反码与补码整理在一起，便于系统性学习和理解它们之间的关系及演变过程。

---
## 七、总结

- **原码**、**反码**与**补码**是有符号整数在计算机中的三种表示方式。
- **补码**因其简化了算术运算和硬件实现，被广泛应用于现代计算机系统中。
- 理解补码的计算方法及其优缺点，对于掌握计算机底层运算机制至关重要。
