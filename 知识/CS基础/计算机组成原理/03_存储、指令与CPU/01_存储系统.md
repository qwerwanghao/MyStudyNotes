# 存储系统

> 这篇笔记要回答一个特别核心的问题：**为什么 CPU 不能直接高效地面对“一个很大的内存”，而必须依赖一整套分层存储系统？**

## 为什么会有“存储系统”这件事

如果只从程序表面去看，你会觉得：
- 变量放在内存里
- CPU 去内存取数据
- 算完再写回

但真实机器里有一个很难绕开的矛盾：

- **CPU 想要的数据访问速度非常快**
- **大容量存储却天然更慢、更便宜**

这意味着：
- 你不能只用“最快的存储”——因为太贵、太小
- 你也不能只用“最大的存储”——因为太慢

所以现代计算机才会设计出一套**分层存储系统**：

- 越靠近 CPU 的层，越快、越小、越贵
- 越远离 CPU 的层，越慢、越大、越便宜

> 存储系统本质上是在做一件事：**用层次结构去缓和“速度、容量、成本”三者之间的冲突。**

## 一个最直观的过程案例：CPU 读取数组元素时发生了什么

假设程序要读一个数组里的值：
- 代码层看起来只是 `a[i]`

但底层真正可能发生的是：
1. CPU 先算出目标地址
2. 先看寄存器里有没有相关地址或中间结果
3. 再看 Cache 里有没有这一段数据
4. 如果 Cache miss，就去主存取
5. 如果相关页还不在主存里，甚至会触发缺页，再去外存调入
6. 数据最终回到更靠近 CPU 的层，然后才真正参与运算

这个过程最能说明一件事：
> 程序表面的一次“读变量”，底层可能牵涉整条存储层次链路。

## 先建立一个核心心智模型

### 存储系统不是“器件名单”，而是“数据离 CPU 远近的阶梯”
从程序执行视角看，不同存储层最大的区别不是名字，而是：

- 数据离 CPU 有多近
- 访问延迟有多高
- 容量有多大
- 成本是否可接受

可以先把它想成一座楼：

1. **寄存器**：就在 CPU 身边，最快，但极少
2. **Cache**：离 CPU 很近，速度快，容量有限
3. **主存（内存）**：容量大得多，但明显更慢
4. **外存（SSD / 硬盘）**：容量最大，但速度最慢

程序“感觉”自己一直在访问内存，
但机器其实一直在想办法让热点数据尽量停留在更靠上的层。

## 为什么分层方案真的可行

因为程序访问数据通常不是完全随机的，而是有规律的。

### 时间局部性
刚访问过的数据，短时间内很可能还会再访问。

### 空间局部性
访问了某个地址后，附近地址也往往很快会被访问。

这就是为什么：
- 循环里的数据容易被重复利用
- 顺序遍历数组通常比离散跳跃访问更友好
- 热点数据可以被更小更快的存储层“兜住”

如果没有局部性，分层存储几乎无法成立。

## 存储层次到底长什么样

## 1. 寄存器
寄存器位于 CPU 内部，是执行过程中最直接、最快的存储位置。

特点：
- 速度最快
- 数量极少
- 通常保存当前正在参与运算的数据、地址、状态

程序员视角：
- 你在高级语言里通常看不到具体寄存器
- 但编译器和 CPU 一直在努力把“最热的数据”留在寄存器里

## 2. Cache
Cache 位于 CPU 与主存之间，用来缓存最近或即将使用的数据块。

特点：
- 比主存快得多
- 比寄存器大，但仍然有限
- 往往分为 L1 / L2 / L3 多级

它存在的意义不是“替代主存”，而是：
- 尽可能减少 CPU 直接等待主存的次数

> Cache 这一层太重要，所以我单独拆成了 [Cache、局部性与存储层次](04_Cache、局部性与存储层次.md) 去讲透。

## 3. 主存（Main Memory）
主存就是我们平时说的“内存”，是 CPU 运行程序时的主要工作区。

特点：
- 容量远大于 Cache
- 速度明显慢于 Cache
- 断电后内容通常丢失

主存不是“无限大背景板”，它有明确限制：
- 带宽有限
- 延迟有限制
- 多个部件可能争用它

## 4. 外存（Secondary Storage）
外存包括 SSD、硬盘等，它们负责长期保存数据。

特点：
- 容量最大
- 成本相对低
- 速度最慢
- 断电后数据仍保留

外存通常不直接参与 CPU 的细粒度执行过程，
但它会在很多关键时刻介入：
- 程序装载
- 文件读写
- 缺页换入换出
- 大规模数据处理

## 主存内部又怎么分

当我们说“主存”时，也不是只有一种器件。

### SRAM
- 用触发器保存数据
- 不需要刷新
- 速度快
- 成本高
- 常用于 Cache

### DRAM
- 用电容保存数据
- 需要周期性刷新
- 成本低、容量大
- 常用于主存

最重要的不是死记定义，而是理解它们为什么分工不同：
- 速度要求极高的层，愿意付出更高成本
- 容量要求极高的层，必须接受更慢一些

## ROM 和“可读不可改”的那一类存储

除了运行期频繁读写的 RAM，机器里还会有一些用于固化程序或配置的存储形式。

常见有：
- ROM
- PROM
- EPROM
- EEPROM
- Flash

这部分最适合理解成：
- 有些数据需要频繁改写
- 有些数据更适合固化、持久、较少改动

所以不同介质不是“谁更先进谁淘汰谁”，而是服务不同目标。

## 为什么讲存储系统不能只停在“分类”

因为真正重要的是：

### 1. 数据如何在各层之间移动
- CPU 不会每次都直接去最底层找数据
- 热数据会尽量停留在更近的层
- 冷数据会留在更远、更大的层

### 2. 每往下一层，代价都会明显上升
- 延迟上升
- 带宽压力增加
- 系统整体吞吐下降

### 3. 程序性能经常不是卡在“算”，而是卡在“等数据”
这也是为什么存储系统会直接影响：
- 数组 vs 链表性能差异
- 顺序访问 vs 跳跃访问
- 单核性能和多核扩展表现

## Cache 为什么值得从这里先提一嘴

虽然 Cache 已经单独成篇，但在《存储系统》里必须先知道它的位置。

### 可以先抓住这三个概念

#### 命中（hit）
要的数据已经在更快的那层里。

#### 缺失（miss）
要的数据不在当前层，需要去下一层取。

#### 存储层次的核心目标
尽可能提高命中率，尽可能减少昂贵的下层访问。

如果你现在只记住一句话，可以记这句：

> **存储系统优化，本质上是在让更多访问停留在更高层。**

## 虚拟存储为什么也必须属于“存储系统”的一部分

很多人第一次学的时候会把虚拟存储误以为只是操作系统内容，
其实从计组视角它也必须放进存储系统里理解。

因为它回答的是：
- 为什么程序看到的是虚拟地址空间
- 为什么物理内存不够时还能继续运行
- 为什么页表、TLB、缺页会影响访问成本

所以在计组路径里，正确的关系是：
- 《存储系统》先讲大框架
- 《虚拟存储、地址转换与页表》再把地址翻译机制讲透

## 常见误区

### 误区 1：内存就是一个统一的大盒子
不是。程序表面看到的是一个整体，底层其实是多层结构。

### 误区 2：CPU 速度快，程序就一定快
不一定。数据送不过来时，CPU 再快也只能等。

### 误区 3：Cache 和虚拟存储只是“额外优化”
它们不是边角装饰，而是现代机器能高效运行的关键机制。

## 读完这篇后，你应该能回答的问题

- 为什么现代计算机必须做分层存储，而不能只有一种存储器？
- 一次简单的数组访问，底层可能经过哪些层？
- 为什么局部性是分层存储能成立的前提？
- 为什么“内存性能”会直接决定程序体验？

## 程序员最该带走的几条直觉

### 1. 大多数性能问题都绕不过访存
CPU 算力提升很快，但数据喂不进去时，再强的执行单元也会空等。

### 2. “数据怎么放”经常和“算法怎么写”一样重要
同样 O(n) 的算法，不同数据布局和访问模式，实际速度可能差很多。

### 3. 存储层次会影响你对程序行为的理解
比如：
- 为什么顺序扫描更快
- 为什么局部变量热点访问更友好
- 为什么多线程共享数据会变复杂

### 4. 以后学操作系统时，很多内存概念都要回到这里
虚拟内存、缺页、页表，不是凭空出现的抽象，而是存储系统向上延伸出来的机制。

## 本章小结

- 存储系统的核心任务，是协调**速度、容量、成本**三者之间的矛盾
- 现代计算机依赖寄存器、Cache、主存、外存形成分层结构
- 分层方案之所以成立，靠的是程序访问中的局部性
- CPU 性能不只取决于“能算多快”，更取决于“能多快拿到数据”
- 后续理解 Cache、虚拟存储、DMA、多核一致性时，都要回到这里的总体框架

## 最小例题

### 例题 1：一次数组访问最理想和最差可能走到哪一层？
假设程序要读取 `a[i]`：

最理想情况：
- 相关数据已经在更高层（比如 Cache）
- CPU 很快就能拿到

较差情况：
- Cache miss，需要去主存

更差情况：
- 对应页当前不在主存，触发缺页，再从外存调入

这个题的关键不在于“精确硬件时序”，而在于理解：
- 一次普通读操作，背后可能跨越多层存储体系

### 例题 2：为什么顺序扫数组常比跳跃访问更友好？
因为顺序访问更符合：
- 空间局部性
- Cache line 预取
- 页和 TLB 的局部利用

## 对比题

### 寄存器 vs Cache vs 主存 vs 外存，最本质的权衡是什么？
请不要只回答“速度 / 容量”。
更好的答案应包含：
- 距离 CPU 的远近
- 延迟差异
- 容量差异
- 成本差异
- 为什么必须分层而不能只留一种

## 参考答案

### 例题 1 参考答案
普通数组访问，最好情况停留在更高层（如 Cache），
差一些要下探到主存，最差甚至可能牵涉外存调页。

### 例题 2 参考答案
顺序访问更符合空间局部性：
- 一次带入的 cache line 更容易继续利用
- 页和 TLB 的局部性也更好

### 对比题参考讲法
- **寄存器**：最快、最少、最靠近执行单元
- **Cache**：速度快但容量有限，兜住热点数据
- **主存**：容量大很多，但延迟高于 Cache
- **外存**：容量最大、速度最慢，用于持久存储和冷数据
分层的根因是：不可能用一种器件同时兼顾速度、容量和成本。

## 常见误答

- 误答 1：把“内存”直接等同于主存，忽略分层体系。
- 误答 2：只会说 Cache 快，却不知道快在哪里、为什么容量小。
- 误答 3：只会背器件分类，不会从访问链角度理解存储系统。

## 能力判定

- **入门**：能列出存储层次组成
- **理解**：能解释为什么必须分层
- **熟练**：能描述一次普通访存的多层路径
- **迁移**：能把局部性和程序性能直接联系起来
