这条路径现在不再只是“平铺 + 大章概述”，而是进一步补齐了 6 个课程级强化专题：

1. Cache、局部性与存储层次

4. 中断、异常与上下文切换
5. DMA、设备控制器与 I/O 协同
6. 多核、缓存一致性与存储模型

这些主题原来在正文里并不是完全没有，而是：
- 要么只是大章中的一个小节
- 要么概念出现了，但没有被真正讲透
- 要么课程里很关键，但现在缺少独立入口

现在这样整理之后，这条线更接近“课程结构”，而不只是“知识点列表”。

## 这个主题最适合解决什么问题

- 为什么二进制、补码、字符编码这些内容是理解程序行为的基础
- CPU、存储、总线、I/O 是如何协作完成一次指令执行的
- 为什么性能、缓存、并行、数据布局会影响实际程序表现
- 为什么很多“高级语言问题”最后会落回到底层表示和执行模型
- 为什么写程序时看起来只是一个变量、一个函数调用，底层却牵涉表示、访存与执行代价
- 为什么现代处理器已经不只是“一个 CPU + 一块内存”这么简单，而是分层、预测、并行和一致性共同作用的系统

## 计组通关清单

如果这条线算真正学到位了，爸爸至少应该能独立讲清下面这些问题：

### 一、系统总览层
- 为什么一台计算机必须同时解决表示、存储、执行、交互四类问题
- 为什么冯·诺依曼模型是整个计组路线的总入口

### 二、数据表示层
- 同一串二进制为什么可以有多种解释
- 为什么补码会成为现代机器的默认有符号整数表示
- 为什么位宽会决定范围、溢出和截断现象

### 三、指令与 CPU 层
- 一条指令从取指到写回的最小执行链是什么
- 指令系统为什么是硬件和软件之间的契约
- 为什么 CPU 的性能不只取决于主频

### 四、存储层
- 为什么现代机器必须做分层存储，而不能只用一种存储器
- Cache、主存、外存分别在解决什么问题
- 为什么局部性会直接决定程序性能

### 五、地址转换层
- 为什么程序不能直接把物理地址当唯一世界
- 页、页框、页表、TLB 分别是什么
- 为什么一次访存可能先变成一次地址翻译

### 六、系统协同层
- 为什么 I/O 不能只靠 CPU 自己一直轮询
- 轮询、中断、DMA 的核心差别是什么
- 为什么总线和互连结构会成为系统瓶颈

### 七、现代处理器理解层
- 为什么现代 CPU 会发展出流水线、预测、乱序和多核
- 为什么“性能问题”很多时候本质上是访存、分支和协同问题

## 能力判定标准

### 初级：能复述
- 能把核心概念按自己的话讲出来
- 不再只会背定义和关键词

### 中级：能举例
- 能自己举出最小例题
- 能解释为什么某个机制是为了解决什么问题出现的

### 进阶：能比较
- 能独立比较：
  - 原码 / 反码 / 补码
  - Cache / TLB
  - 轮询 / 中断 / DMA
  - RISC / CISC
- 能说清差异不是“名字不同”，而是设计取舍不同

### 熟练：能迁移
- 能把计组知识迁移到程序性能、系统协同、调试现象里
- 看到问题时，会自然想到：
  - 是表示问题？
  - 是访存问题？
  - 是控制流问题？
  - 是协同瓶颈问题？

> 如果这些问题你已经能不用看笔记、自己顺着讲出来，而且还能举例、比较、迁移，这条计组主线就已经真正开始变成你的底层能力了。
