# 计算机的数据表示

> 这篇笔记的目标是回答：**现实世界里的数字、字符、布尔值、地址，为什么最终都能变成机器里的 0 和 1？**

## 为什么“数据表示”是计组里的核心入口

因为后面你学到的一切几乎都建立在这里：
- 指令为什么能被编码
- 整数为什么会溢出
- 字符为什么能进入机器
- 地址为什么能被运算

如果不先理解“表示规则”，后面很多内容都只能靠死记。

## 先建立一个核心心智模型

### 机器里没有“意义”，只有编码
同一串二进制数据，本身并不会告诉你：
- 它是数字
- 它是字符
- 它是地址
- 它是指令

真正决定意义的是：
- 位宽是多少
- 有没有符号位
- 解释规则是什么
- 使用场景是什么

> 所以“数据表示”的核心不是 bit 本身，而是 **bit 如何被解释**。

## 为什么二进制是机器世界的基础

现实硬件更容易稳定地区分两种状态：
- 高 / 低
- 开 / 关
- 有电 / 无电

所以把所有信息最终压缩成 0 和 1，是一种很自然的工程选择。

二进制的优势：
- 容易可靠表示
- 便于逻辑运算
- 适合组合出更复杂的信息结构

## 常见的数据表示类别

## 1. 无符号整数
最直接的一类表示。

它回答的是：
- 这串 bit 如果只看大小，能表示多大的非负整数？

例如 8 位时：
- 全 0 表示最小值
- 全 1 表示最大值

这种表示方式最适合：
- 计数
- 大小
- 索引
- 原始字节值

## 2. 有符号整数
一旦你想表示负数，就必须回答：
- 负号怎么编码进 bit 序列里？

这就引出了：
- 原码
- 反码
- 补码

其中真正被现代机器广泛采用的是补码，这部分会在 [原码、反码与补码](03_原码、反码与补码.md) 里展开。

## 3. 浮点数
整数只能表示离散数值。

但现实计算经常需要：
- 小数
- 很大或很小的数
- 科学计数形式

这时就需要浮点表示。

最重要的不是先记 IEEE 754 细节，而是理解：
- 浮点数不是“无限精确的小数”
- 它本质上是对实数的有限近似表示

所以才会有：
- 精度误差
- 舍入问题
- 比较陷阱

## 4. 字符与文本
字符本质上也是一种“数据表示”。

不同点在于：
- 这里的 bit / byte 被解释成符号而不是数值大小

这部分和 [计算机中的字符](../01_系统入门与信息基础/03_计算机中的字符.md) 强相关。

## “位宽”为什么一定要一起理解

任何表示都离不开位宽。

因为位宽决定：
- 能表示多少种状态
- 数值范围多大
- 是否容易溢出
- 编码布局如何安排

例如：
- 8 位和 32 位表达能力完全不同
- 同一种规则在不同位宽下，范围也不同

## 数据表示最容易踩的坑

### 1. 以为“数据本身自带意义”
其实意义来自解释规则。

### 2. 以为“位模式一样，数值一定一样”
不一定。
- 同一位模式可能被解释成不同类型

### 3. 以为“表示”和“运算”是分开的
其实不分开。
- 你怎么表示，往往直接决定你怎么运算最方便

这也是为什么补码最终胜出：
它和硬件加法器更匹配。

## 程序员最该带走的几个直觉

### 1. 一切高级类型最终都要落到底层表示

### 2. 表示方式会影响范围、精度、溢出和性能

### 3. 同一串 bit 不是“有一个真相”，而是“有多种可能解释”

### 4. 真正理解表示，才能理解为什么底层 bug 会长成那个样子

## 本章小结

- 数据表示的核心问题是：bit 序列如何被赋予规则和意义
- 二进制之所以成为基础，是因为硬件易于稳定实现两态表示
- 整数、浮点、字符本质上都是不同的数据解释规则
- 表示方式不仅决定“能存什么”，也决定“怎么更方便地算”

## 最小例题

### 例题 1：为什么 `11111111` 不能脱离解释规则直接说“它就是某个固定值”？
因为它可能被解释成：
- 无符号整数 255
- 8 位补码 -1
- 字符编码的一部分
- 指令字段的一部分

### 例题 2：为什么位宽变化会改变可表示范围？
因为位宽决定了总状态数。位越多，可区分的模式越多，范围自然扩大。

## 对比题

### “位模式” vs “值” vs “语义”，这三者最关键的区别是什么？
请说明：
- 位模式是什么
- 值在什么规则下出现
- 语义又如何依赖上下文

## 参考答案

### 例题 1 参考答案
关键不是去找一个唯一值，而是先问：
- 当前按什么规则解释？
如果解释规则不同，值自然就不同。

### 例题 2 参考答案
位宽增加，本质上就是可用位模式数量增加，因此可表示范围也随之扩大。

### 对比题参考讲法
- **位模式**：一串原始 bit
- **值**：在某个表示规则下得到的数学结果
- **语义**：在程序 / 协议 / 系统上下文中，这个值具体代表什么

## 常见误答

- 误答 1：把位模式直接等同于值。
- 误答 2：默认数据天然自带唯一意义。

## 能力判定

- **入门**：知道 bit 序列要靠规则解释
- **理解**：能说出位宽为什么影响范围
- **熟练**：能分清位模式、值、语义三层
- **迁移**：能把这种理解用到整数、字符、地址和指令上

## 通关检查点

学完这篇后，你至少应该能做到：
- 说清数据表示为什么不是“存储细节”，而是计组基础
- 解释为什么同一串 bit 可以有多重解释
- 理解位宽和表示范围之间的关系
