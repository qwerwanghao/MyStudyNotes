## 为什么 IP 地址不能只背成“主机编号”

IP 地址不只是给设备起名字，它还承担：
- 标识一个网络位置
- 决定本机是否认为对端在同一网段
- 决定是直接二层发送，还是先交给网关

## 先建立一个核心心智模型

### IP 地址 = 网络前缀 + 主机部分
真正关键的不是“192.168.1.10 长什么样”，而是：
- 哪一部分代表网络
- 哪一部分代表主机

这就要靠子网掩码或 CIDR 前缀长度来定义。

## CIDR 为什么重要

CIDR 的核心价值是：
- 用前缀长度表示网络范围
- 比传统类地址更灵活
- 更适合真实路由聚合与网络规划

比如 `/24` 的重点不是形式，而是：
- 前 24 位用于标识网络
- 剩余位用于主机区分

## 一个最小过程案例：为什么同一机器会判断“这个目标需要先找网关”

当主机要发送给某个目标 IP 时，会先做一件事：
- 用自己的地址和子网规则判断：对方是不是本地网段

如果是：
- 尝试在本地链路直接找到对方

如果不是：
- 交给默认网关继续转发

这个判断的根源，就是 IP 地址与前缀规则。

## 常见误区

### 误区 1：IP 地址就是设备唯一编号
不完整。IP 更准确地说是网络层位置标识，而且可能变化。

### 误区 2：子网掩码只是记忆题
错误。它直接决定本机如何判断本地与远端。

### 误区 3：同一网段就一定能通信，跨网段就一定不能
也不绝对。跨网段能否通信取决于是否有正确网关与路由路径。

## 最小例题

### 例题 1：为什么同样是发数据，有时直接找对方，有时先找网关？
答案要点：
- 因为主机会先根据 IP 和前缀判断目标是否属于本地网段

### 例题 2：为什么 CIDR 比传统类地址更实用？
答案要点：
- 因为它能更灵活表达网络前缀
- 更适合现实网络规划与路由聚合

## 对比题

### IP 地址中的“位置”属性 vs “身份”属性，哪个更接近网络层本质？为什么？

## 参考答案

### 例题 1 参考答案
主机会先根据本机地址和子网规则判断目标是否本地可达。本地就直接在局域网里找，非本地就交给网关。

### 例题 2 参考答案
CIDR 更实用，因为它让网络前缀定义更灵活，避免类地址过于僵硬，更适合现实网络划分与聚合。

### 对比题参考讲法
IP 更接近“网络位置标识”，而不是永久身份标签。网络层关心的是怎么把包送到那个位置。

## 常见误答

- “IP 地址永远等同于设备身份。”
- “子网掩码只是为了考试算主机数。”
- “跨网段通信和主机没关系，全是路由器问题。”
