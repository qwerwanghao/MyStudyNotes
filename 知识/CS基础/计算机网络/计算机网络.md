## 计算机网络通关清单

如果这条线算真正学到位了，爸爸至少应该能独立讲清下面这些问题：

### 一、分层与通信全景层
- 为什么网络必须分层
- 为什么应用一次发送不等于链路一次传输
- 为什么 MTU 和封装会影响真实通信表现

### 二、IP 与路由层
- 为什么 IP 更像网络位置而不是永久身份
- 为什么主机会判断本地网段与远端网段
- 为什么路由更像逐跳决策而不是全路径脚本
- NAT、TTL、分片分别在解决什么现实问题

### 三、链路与局域网层
- 为什么知道 IP 还不够，还需要 MAC 与 ARP
- 为什么默认网关是跨网段通信关键入口
- 为什么交换机、广播域和 VLAN 需要同时理解

### 四、传输层
- TCP 和 UDP 为什么是不同服务模型，而不是简单高低级关系
- 三次握手、四次挥手在确认什么
- 为什么可靠传输离不开编号、确认、窗口与重传
- 为什么丢包会放大成延迟与拥塞问题

### 五、应用层与安全层
- HTTP 为什么不能只当成文本报文格式来学
- DNS 为什么是多层解析与缓存体系
- 为什么 HTTPS 不只是“加密一下”
- TCP 建连与 TLS 握手分别在建立什么

### 六、性能与工程层
- RTT、带宽、吞吐为什么不是一个意思
- 为什么网络优化很多时候是在优化路径而不是换协议
- 为什么 CDN、代理、缓存与入口组织会改变用户体验
- 为什么抓包是协议理解真正落地的关键工具

### 七、综合排障层
- 从 URL 到页面返回会经历哪些阶段
- 为什么页面慢和接口慢都必须拆阶段排查
- 超时、拒绝连接、连接重置分别更像哪类失败

## 能力判定标准

### 初级：能复述
- 能用自己的话解释分层、IP、TCP、HTTP、DNS、TLS、拥塞和抓包等核心概念

### 中级：能举例
- 能举出最小过程案例，如：
  - 为什么访问外网时常先找网关 MAC
  - 为什么大包不一定是一大帧
  - 为什么页面慢不一定是服务端慢
  - 为什么一个连接会重置而不是超时

### 进阶：能比较
- 能独立比较：
  - IP vs MAC
  - TCP vs UDP
  - 流量控制 vs 拥塞控制
  - HTTP vs HTTPS
  - 递归 DNS vs 迭代解析
  - 超时 vs 拒绝连接 vs 连接重置

### 熟练：能迁移
- 能把网络知识迁移到：
  - 后端接口与网关设计
  - 服务稳定性分析
  - 性能瓶颈定位
  - 高并发 I/O 与抓包排障
  - CDN / 代理 / 多地域部署理解

## 这个主题最适合回答什么问题

- 数据是如何从一台机器传到另一台机器的
- 为什么会有握手、重传、超时、丢包、拥塞这些现象
- 为什么同样是“请求接口”，延迟和稳定性会差很多
- 为什么应用层问题经常要回到底层协议来看
- 为什么“能访问”不等于“网络链路设计合理、性能足够好”
