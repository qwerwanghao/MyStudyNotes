# DHCP、网关与局域网接入

## 为什么“自动联网”不是理所当然

设备刚接入网络时，其实最开始什么都不知道：
- 自己的 IP 是什么
- 子网范围是什么
- 默认网关是谁
- DNS 该问谁

如果这些配置都靠手工填，网络维护会非常重。
所以 DHCP 的现实意义是：
> **把接入阶段的关键网络配置自动发给新主机。**

## 先建立一个核心心智模型

### DHCP 不是只发 IP，而是在发“一整套能开始通信的最小配置”
典型内容包括：
- IP 地址
- 子网掩码 / 前缀
- 默认网关
- DNS 服务器
- 租约时间

## 默认网关为什么总会一起出现

因为即使主机拿到了本地地址，
如果不知道：
- 非本地目标该交给谁

它依然无法正常访问更大的网络。
所以网关配置是“能出局域网”的关键。

## 一个最小过程案例：为什么“能拿到 IP”不等于“一定能正常上网”

假设主机成功获得一个 IP，
但如果：
- 没有正确网关
- 或没有正确 DNS

结果可能是：
- 能和局域网内部分主机通信
- 但访问外网失败
- 或只能访问 IP，不能访问域名

这个例子说明：
- 联网配置是成套生效的，不是只看有没有地址

## 常见误区

### 误区 1：DHCP 只负责发一个 IP 地址
不完整。它常常负责发整套关键网络参数。

### 误区 2：只要有 IP，外网就一定能通
错误。还要看网关、DNS 甚至路由策略。

### 误区 3：默认网关就是某个互联网网站地址
不是。默认网关是当前本地网络里负责把流量继续往外送的下一跳设备。

## 最小例题

### 例题 1：为什么一台主机“拿到 IP”后仍可能不能正常访问网站？
答案要点：
- 因为还可能缺少正确的网关或 DNS 配置

### 例题 2：为什么默认网关是主机跨网段通信的关键配置？
答案要点：
- 因为主机不知道外部全网路径
- 非本地目标通常要先交给默认网关

## 对比题

### “本地地址配置” vs “跨网段出站能力配置”，最本质分别由什么决定？

## 参考答案

### 例题 1 参考答案
拿到 IP 不等于能上网站，因为网站访问通常还依赖正确网关和 DNS。少任何一个，网络体验都可能不完整。

### 例题 2 参考答案
默认网关关键，是因为主机通常只认识本地网段；跨网段流量必须先交给本地更懂外部路径的设备处理。

### 对比题参考讲法
- 本地地址配置决定主机在本地网络里的身份与范围
- 网关 / DNS 等配置决定它能否顺利走向更大的网络世界

## 常见误答

- “DHCP 只要发了 IP，其他都是可选。”
- “默认网关就是离我最近的网站服务器。”
- “有 IP 不能访问网站，只可能是网站坏了。”
