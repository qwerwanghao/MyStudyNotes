# MAC、ARP与局域网寻址

## 为什么只有 IP 还不够

网络层解决的是“送到哪台主机或哪张网络位置”，
但在本地链路真正发出去时，还要回答：
- 当前这一跳应该发给哪个链路层目的地

这就要落到：
- MAC 地址
- ARP 解析

## 先建立一个核心心智模型

### IP 决定“逻辑上找谁”，MAC 决定“这一跳帧发给谁”
可以先这样理解：
- IP 是网络层定位
- MAC 是当前局域网这一跳的链路层接收目标

它们不是二选一，而是不同层协作。

## ARP 为什么存在

当主机已经知道：
- 目标 IP 在本地网段
或
- 下一跳网关 IP 是谁

仍然还要知道：
- 对应的 MAC 地址是什么

ARP 的作用就是：
- 在局域网里把某个 IP 和当前可达的 MAC 关系对应起来

## 一个最小过程案例：为什么访问外网时 ARP 解析的往往不是网站服务器，而是网关

如果目标不在本地网段：
- 主机不会直接去找远端服务器的 MAC
- 而是先找默认网关的 MAC
- 当前帧发给网关，由网关继续转发

这个例子特别重要，因为它说明：
- 局域网里的链路层目标，未必等于最终应用层目标

## ARP 广播为什么是“局域网问题”

ARP 常见做法是：
- 在当前广播域里问“谁是这个 IP”

这说明它天然依赖：
- 本地链路范围
- 广播可达性

所以 ARP 不是全网协议，而是本地网络寻址工具。

## 常见误区

### 误区 1：知道目标 IP 就能直接发过去
不对。当前一跳链路还需要知道该发给哪个 MAC。

### 误区 2：访问外网时，主机会直接解析远端网站的 MAC 地址
错误。主机通常只需要本地下一跳（网关）的 MAC。

### 误区 3：MAC 地址决定了跨互联网完整路径
不是。MAC 主要在当前链路 / 当前广播域里有意义。

## 最小例题

### 例题 1：为什么访问同一局域网主机和访问公网网站，ARP 解析目标可能不同？
答案要点：
- 本地网段时，可能直接解析对方主机 MAC
- 跨网段时，常解析默认网关 MAC

### 例题 2：为什么 ARP 说到底是“本地网络”问题？
答案要点：
- 因为它依赖广播域
- 解决的是本地链路中的地址映射，而不是全网寻址

## 对比题

### IP 寻址 vs MAC 寻址，最本质分别在解决哪一层问题？

## 参考答案

### 例题 1 参考答案
同局域网时，主机可能直接找目标主机 MAC；跨网段时，它只需要先把帧发给网关，因此 ARP 解析对象常变成网关。

### 例题 2 参考答案
ARP 本质上是本地网络问题，因为它依赖当前广播域，用来完成链路层一跳内的地址映射，而不是全网定位。

### 对比题参考讲法
- IP 寻址：决定网络层目标位置
- MAC 寻址：决定当前链路帧发给谁

## 常见误答

- “ARP 是全网查找协议。”
- “MAC 地址比 IP 地址更底层，所以能决定整个路径。”
- “访问网站时会先 ARP 到网站服务器。”
