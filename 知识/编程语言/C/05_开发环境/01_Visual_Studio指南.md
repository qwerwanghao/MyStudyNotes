# Visual Studio 创建工程指南

在使用 Visual Studio 创建和管理工程时，有几个重要的注意事项，以确保项目结构清晰，易于维护和管理。

## 1. 创建工程

- 选择“空项目”选项，以便从头开始设置项目。
- 确保不使用预编译头文件（Precompiled Headers），以保持项目的简洁性和灵活性。

## 2. 目录结构

为了保持项目的清晰和可管理性，建议将目录分开保存：

- **编译目录**: 保存编译生成的中间文件和输出文件。示例：`project.win32`
- **源码目录**: 保存源代码文件。示例：`src`
- **运行时目录**: 保存运行时生成的文件，如可执行文件和日志文件。示例：`bin`

## 3. 配置调试器

- 设置调试器的工作目录为运行时目录，以确保调试器在运行程序时使用正确的工作目录。

## 4. 源码管理

- 手动在源码目录中创建源文件，并将其添加到 Visual Studio 项目中。
- 注意保持源码路径和文件夹分类的一致性。

## 5. 文件筛选器

- 在 Visual Studio 中使用筛选器来组织项目中的文件。
- 确保筛选器结构与源码目录结构保持一致，以便于项目管理。

## 示例目录结构

- **编译目录**: `project.win32`
- **源码目录**: `src`
- **运行时目录**: `bin`

这种结构有助于保持项目的整洁性，简化管理和维护工作。
