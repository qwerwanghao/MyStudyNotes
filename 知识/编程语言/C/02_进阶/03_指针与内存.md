# C语言指针与内存

## 指针与内存

### 指针基础
- **定义**：指针是存储内存地址的变量
- **声明语法**：`类型 *指针名;`
- **重要概念**：
  - 取地址运算符 `&`
  - 解引用运算符 `*`
  - NULL指针

```c
int x = 10;
int *p = &x;    // 正确：将x的地址赋给指针p
int *p1;        // 警告：未初始化的指针很危险
p1 = NULL;      // 正确：将指针初始化为NULL
```

### 指针与数组
⚠️ **常见误解**：数组和指针不完全等价

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;  // 数组名在大多数情况下会退化为指针
// 但以下情况不同：
sizeof(arr);   // 返回整个数组的大小：5 * sizeof(int)
sizeof(p);     // 返回指针的大小：通常是4或8字节
```

### 动态内存管理
```c
// 内存分配最佳实践
void *ptr = malloc(size);
if (ptr == NULL) {
    // 错误处理
    return;
}
// 使用内存
free(ptr);     // 防止内存泄漏
ptr = NULL;    // 防止悬挂指针
```

#### malloc vs. calloc
| 特性 | malloc | calloc |
|------|---------|---------|
| 初始化 | 不初始化 | 初始化为0 |
| 参数个数 | 1个 | 2个 |
| 性能 | 较快 | 较慢 |
| 适用场景 | 一般分配 | 需要清零时 |

### 常见错误和调试技巧

#### 1. 段错误(Segmentation Fault)常见原因
- 解引用NULL指针
- 访问已释放的内存
- 数组越界
- 栈溢出

#### 2. 调试技巧
```c
// 打印指针地址和值
printf("指针地址: %p\n", (void*)ptr);
printf("指针指向的值: %d\n", *ptr);

// GDB调试命令
/*
break main    // 设置断点
print ptr     // 打印指针值
print *ptr    // 打印指针指向的值
*/
```

## 实际应用示例

### 动态数组实现
```c
typedef struct {
    int *data;
    size_t size;
    size_t capacity;
} DynamicArray;

DynamicArray* createArray(size_t initialCapacity) {
    DynamicArray *arr = malloc(sizeof(DynamicArray));
    if (!arr) return NULL;
    
    arr->data = malloc(initialCapacity * sizeof(int));
    if (!arr->data) {
        free(arr);
        return NULL;
    }
    
    arr->size = 0;
    arr->capacity = initialCapacity;
    return arr;
}
```

## 面试高频考点

### 指针与引用的区别
1. 指针可以为NULL，引用必须初始化
2. 指针可以更改指向，引用不能
3. 指针需要解引用，引用直接使用

### 内存泄漏检查工具
- Valgrind
- AddressSanitizer
- Dr.Memory
