| `src`      | `source`         | 源           |
| `func`     | `function`       | 函数         |
| `lhs`      | `left hand side` | 左手边       |

## 二、C语言编码规范

### 1. Linux 命名法
- **描述**: 两个单词之间用 `_` 分隔。
- **示例**:
  ```c
  void test_func() {
  }
  ```

### 2. 驼峰命名法
- **描述**: 常用于 Java 和游戏引擎中。
- **示例**:
  ```c
  void testFunc() {
  }
  ```

### 3. 匈牙利命名法
- **描述**: 微软常用的命名法。
- **示例**:
  ```c
  void TestFunc() {
  }
  ```

## 三、常见错误

1. **丢失分号**: 每句代码习惯性地加上分号。
2. **大括号未能匹配**: 预防措施是开始时就先将括号打好。
3. **小括号不匹配**: 预防措施是开始时将所有括号打好。
4. **未定义的变量/重复定义了一个变量**: 使用了一个变量但找不到定义。
5. **使用一个没有初始化的变量**: 变量里存放的数据不可控，必须重视此警告。
   - 明确每个变量的作用。
6. **链接错误**: 找不到函数的实现或符号的实现。
7. **类型不匹配**: 确保变量类型一致。
8. **超出预期的死循环**: 避免死循环的出现。
9. **预防代码错误的最佳方法**: 写一行代码后，脑中执行一次。
10. **学会阅读编译器的错误信息**: 理解错误信息，有助于快速定位问题。
