## 专题之间怎么衔接

- 基础知识 → 面向对象：从语法理解过渡到代码组织能力
- 面向对象 → 高级特性：从类设计进入异步、线程、反射、模式等工程问题
- 高级特性 → Unity 基础：把语言能力投入具体引擎工作流
- Unity 基础 → Unity 进阶：从“会搭东西”进入“会管理系统复杂度”
- Unity 进阶 → 大型游戏开发实践：从技术系统走向项目交付、维护和协作

## 适合带着哪些问题来读

- C# 的语言能力应该先补到什么程度，才能做 Unity 开发
- 面向对象、委托、事件、泛型、异步这些概念怎么串起来
- Unity 项目从“能跑”到“可维护”之间差了哪些工程实践
- 游戏开发里哪些问题应该回到语言层面解决，哪些应该回到引擎层面解决
- 如果我要做长期维护的项目，应该从哪个阶段开始关注框架、资源、热更新和发布
