# Unity状态管理

<!-- DEEP-TOPIC-GUIDE-START -->
> 状态管理是项目复杂度上升后最容易决定“系统是否还能控住”的专题之一。很多表面上的功能问题，其实根源都是状态边界没设计清楚。

## 先读这篇时，建议带着这些问题

- 状态到底应该放在哪一层：全局、场景、角色、UI 还是系统内部
- 简单状态机和分层状态机分别适合什么复杂度
- 状态切换为什么经常和动画、AI、UI、保存系统纠缠在一起
- 为什么项目越大，状态管理越像一套约束机制

## 推荐阅读顺序

1. 状态管理概述
2. 基础状态机
3. 角色状态机
4. 分层状态机
5. 状态数据管理
6. 状态同步与持久化

## 这一页最值得重点抓住的概念

- **状态管理的核心不是切换，而是边界与一致性**
- **状态层级越复杂，越需要清楚划分职责**
- **状态系统会直接影响动画、AI、UI 和存档设计**
- **如果状态流向不清晰，后期 bug 会非常难追**

## 1. 状态管理概述

### 1.1 什么是状态管理

状态管理是指在游戏开发中对各种游戏对象状态的管理，包括：

- 游戏全局状态
- 场景状态
- 角色状态
- UI状态
- 游戏进度状态

### 1.2 状态管理的重要性

- 维护游戏逻辑的一致性
- 简化状态转换的复杂度
- 提高代码的可维护性
- 方便调试和错误追踪

## 2. 状态机系统

### 2.1 基础状态机

```csharp
public abstract class State
{
    public abstract void Enter();
    public abstract void Update();
    public abstract void Exit();
}

public class StateMachine
{
    private State currentState;
    
    public void ChangeState(State newState)
    {
        if (currentState != null)
        {
            currentState.Exit();
        }
        
        currentState = newState;
        currentState.Enter();
    }
    
    public void Update()
    {
        if (currentState != null)
        {
            currentState.Update();
        }
    }
}
```

### 2.2 角色状态机

```csharp
public class PlayerController : MonoBehaviour
{
    private StateMachine stateMachine;
    private IdleState idleState;
    private RunState runState;
    private JumpState jumpState;
    private AttackState attackState;
    
    void Start()
    {
        stateMachine = new StateMachine();
        
        // 初始化状态
        idleState = new IdleState(this);
        runState = new RunState(this);
        jumpState = new JumpState(this);
        attackState = new AttackState(this);
        
        // 设置初始状态
        stateMachine.ChangeState(idleState);
    }
    
    void Update()
    {
        stateMachine.Update();
    }
}

public class IdleState : State
{
    private PlayerController player;
    
    public IdleState(PlayerController player)
    {
        this.player = player;
    }
    
    public override void Enter()
    {
        // 播放待机动画
        player.GetComponent<Animator>().Play("Idle");
    }
    
    public override void Update()
    {
        // 检测状态转换条件
        if (Input.GetKey(KeyCode.W) || Input.GetKey(KeyCode.A) || 
            Input.GetKey(KeyCode.S) || Input.GetKey(KeyCode.D))
        {
            player.GetComponent<StateMachine>().ChangeState(new RunState(player));
        }
    }
    
    public override void Exit()
    {
        // 清理状态
    }
}
```

## 3. 分层状态机

### 3.1 分层状态机实现

```csharp
public class HierarchicalStateMachine
{
    private Dictionary<string, State> states = new Dictionary<string, State>();
    private Stack<State> stateStack = new Stack<State>();
    
    public void PushState(string stateName)
    {
        if (states.TryGetValue(stateName, out State state))
        {
            if (stateStack.Count > 0)
            {
                stateStack.Peek().Pause();
            }
            
            stateStack.Push(state);
            state.Enter();
        }
    }
    
    public void PopState()
    {
        if (stateStack.Count > 0)
        {
            State state = stateStack.Pop();
            state.Exit();
            
            if (stateStack.Count > 0)
            {
                stateStack.Peek().Resume();
            }
        }
    }
    
    public void Update()
    {
        if (stateStack.Count > 0)
        {
            stateStack.Peek().Update();
        }
    }
}

public abstract class HierarchicalState : State
{
    public virtual void Pause() { }
    public virtual void Resume() { }
}
```

### 3.2 游戏状态管理

```csharp
public class GameStateManager : MonoBehaviour
{
    private HierarchicalStateMachine stateMachine;
    
    private void Start()
    {
        stateMachine = new HierarchicalStateMachine();
        
        // 注册游戏状态
        stateMachine.RegisterState("MainMenu", new MainMenuState());
        stateMachine.RegisterState("Playing", new PlayingState());
        stateMachine.RegisterState("Paused", new PausedState());
        stateMachine.RegisterState("GameOver", new GameOverState());
        
        // 进入初始状态
        stateMachine.PushState("MainMenu");
    }
    
    private void Update()
    {
        stateMachine.Update();
    }
}

public class MainMenuState : HierarchicalState
{
    public override void Enter()
    {
        // 显示主菜单UI
        UIManager.Instance.ShowPanel("MainMenu");
    }
    
    public override void Update()
    {
        // 处理主菜单逻辑
    }
    
    public override void Exit()
    {
        // 隐藏主菜单UI
        UIManager.Instance.HidePanel("MainMenu");
    }
}
```

public class GameState
{
    public PlayerState playerState;

    {
        string json = JsonUtility.ToJson(this);

        if (PlayerPrefs.HasKey("GameState"))

### 4.2 状态观察者模式

```csharp
public class StateObserver : MonoBehaviour
{
    public delegate void StateChangeHandler(object data);
    private Dictionary<string, StateChangeHandler> observers = 
        new Dictionary<string, StateChangeHandler>();
    
    public void RegisterObserver(string stateName, StateChangeHandler handler)
    {
        if (!observers.ContainsKey(stateName))
        {
            observers[stateName] = handler;
        }
        else
        {
            observers[stateName] += handler;
        }
    }
    
    public void UnregisterObserver(string stateName, StateChangeHandler handler)
    {
        if (observers.ContainsKey(stateName))
        {
            observers[stateName] -= handler;
        }
    }
    
    public void NotifyStateChange(string stateName, object data)
    {
        if (observers.TryGetValue(stateName, out StateChangeHandler handler))
        {
            handler?.Invoke(data);
        }
    }
}
```

## 5. 状态同步

### 5.1 网络状态同步

```csharp
public class NetworkStateSync : MonoBehaviour
{
    private NetworkView networkView;
    private GameState gameState;
    
    void Start()
    {
        networkView = GetComponent<NetworkView>();
        gameState = new GameState();
    }
    
    void OnSerializeNetworkView(BitStream stream, NetworkMessageInfo info)
    {
        if (stream.isWriting)
        {
            // 发送状态
            Vector3 pos = transform.position;
            stream.Serialize(ref pos);
        }
        else
        {
            // 接收状态
            Vector3 pos = Vector3.zero;
            stream.Serialize(ref pos);
            transform.position = pos;
        }
    }
    
    [RPC]
    void SyncGameState(string stateJson)
    {
        JsonUtility.FromJsonOverwrite(stateJson, gameState);
        // 更新游戏状态
        UpdateGameState();
    }
}
```

### 5.2 状态插值

```csharp
public class StateInterpolation : MonoBehaviour
{
    private Vector3 lastPosition;
    private Vector3 targetPosition;
    private float interpolationTime = 0.1f;
    private float currentTime = 0;
    
    void Update()
    {
        if (currentTime < interpolationTime)
        {
            currentTime += Time.deltaTime;
            float t = currentTime / interpolationTime;
            
            // 使用平滑插值
            transform.position = Vector3.Lerp(lastPosition, targetPosition, t);
        }
    }
    
    public void UpdateTargetPosition(Vector3 newPosition)
    {
        lastPosition = transform.position;
        targetPosition = newPosition;
        currentTime = 0;
    }
}
```

## 6. 状态持久化

### 6.1 状态序列化

```csharp
public class StatePersistence
{
    public static void SaveState(GameState state, string filename)
    {
        string json = JsonUtility.ToJson(state);
        string path = Path.Combine(Application.persistentDataPath, filename);
        
        try
        {
            File.WriteAllText(path, json);
            Debug.Log($"状态已保存到: {path}");
        }
        catch (Exception e)
        {
            Debug.LogError($"保存状态失败: {e.Message}");
        }
    }
    
    public static GameState LoadState(string filename)
    {
        string path = Path.Combine(Application.persistentDataPath, filename);
        
        try
        {
            if (File.Exists(path))
            {
                string json = File.ReadAllText(path);
                return JsonUtility.FromJson<GameState>(json);
            }
        }
        catch (Exception e)
        {
            Debug.LogError($"加载状态失败: {e.Message}");
        }
        
        return new GameState();
    }
}
```

### 6.2 状态回滚

```csharp
public class StateRollback
{
    private Queue<GameState> stateHistory = new Queue<GameState>();
    private int maxHistorySize = 10;
    
    public void SaveStateSnapshot()
    {
        GameState currentState = new GameState();
        // 保存当前状态
        stateHistory.Enqueue(currentState);
        
        if (stateHistory.Count > maxHistorySize)
        {
            stateHistory.Dequeue();
        }
    }
    
    public GameState RollbackToPreviousState()
    {
        if (stateHistory.Count > 0)
        {
            return stateHistory.Dequeue();
        }
        return null;
    }
}
```

## 7. 最佳实践

### 7.1 状态设计原则

1. 单一职责
   - 每个状态类只负责一个具体的状态
   - 状态逻辑清晰明确

2. 状态转换
   - 明确定义状态转换条件
   - 避免状态转换死循环
   - 保持状态转换的可追踪性

3. 性能优化
   - 避免频繁的状态切换
   - 合理使用状态缓存
   - 优化状态更新逻辑

### 7.2 调试技巧

1. 状态监控
   - 实时显示当前状态
   - 记录状态转换历史
   - 状态数据可视化

2. 错误处理
   - 状态异常检测
   - 状态恢复机制
   - 日志记录
