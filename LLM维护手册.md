# LLM维护手册

## 目标

LLM 在这个仓库里的职责不是“代替我思考”，而是：

- 降低记录阻力
- 把碎片整理成结构化笔记
- 维护索引、链接和状态
- 帮我做周期性回顾

## 允许做的事

- 根据 `收集/Inbox.md` 和 `日志/` 整理当天内容
- 新建知识笔记、感悟笔记、总结草稿、计划草稿
- 合并明显重复的笔记
- 给笔记补充 frontmatter、标签、相关链接
- 把零散条目归档到更合适的位置

## 不允许做的事

- 擅自删除原始记录
- 把不确定的知识写成确定事实
- 把我的情绪、观点、美化成并不存在的“深刻结论”
- 大规模重命名或迁移现有目录，除非我明确要求

## 笔记类型

- `capture`：原始碎片，通常放在 `收集/`
- `daily`：每日记录，通常放在 `日志/`
- `knowledge`：长期知识笔记，通常放在 `知识/`
- `insight`：个人判断、体会、方法论，通常放在 `感悟/`
- `review`：周、月、年、阶段复盘，通常放在 `总结/`
- `plan`：阶段规划、年度规划，通常放在 `展望/`

## 建议 frontmatter

```yaml
---
title: 标题
type: daily | capture | knowledge | insight | review | plan
date: 2026-04-12
status: raw | distilled | evergreen
tags: []
source: []
related: []
---
```

规则：

- `raw` 用于原始记录和未整理内容
- `distilled` 用于已经梳理过但还会继续修改的内容
- `evergreen` 用于长期稳定的知识笔记

## 日常维护流程

### 每天

1. 读取 `收集/Inbox.md` 和当天 `日志/`
2. 把明显属于长期知识的内容拆到 `知识/`
3. 把明显属于个人判断的内容拆到 `感悟/`
4. 给当天日志补充“待跟进问题”和“可沉淀主题”
5. 如有必要，更新 [索引/工作台.md](索引/工作台.md) 中的当前关注项

### 每周

1. 汇总最近 7 天日志
2. 生成周回顾草稿到 `总结/`
3. 找出重复出现的问题、主题、目标
4. 标记值得长期沉淀的主题

### 每月 / 每年

1. 基于日志和周回顾生成阶段总结
2. 把阶段总结和未完成事项转成下一阶段计划
3. 保留原始材料，不覆盖旧判断

## 新建还是更新

优先更新已有笔记，只有在下列情况下新建：

- 新主题和现有笔记边界明显不同
- 原笔记已经过长，不再适合继续扩展
- 需要把原始记录升级成长期笔记

## 命名建议

- 日志：`YYYY-MM-DD.md`
- 感悟：`MM.DD感悟.md` 或更清晰的主题名
- 总结：`2026年总结.md`、`2026-W15周回顾.md`
- 计划：`2026年计划.md`、`第二季度展望.md`
- 知识：尽量直接用主题名，不加“笔记”“总结”等空词

## 链接原则

- 长期笔记之间尽量互相链接
- 日志里只链接当天真正涉及的知识和感悟
- 索引页只放入口，不堆细节正文

## 知识目录约束

- `知识/` 下优先使用 `大类 / 主题 / 分组 / 文件` 结构
- 不再新增 `Content/`、`Media/` 这类中间目录
- 图片和示意图统一放在同级 `资源/`
- 当路径层级继续变深时，优先改文件名或补索引页，而不是继续套目录

## 遇到不确定内容时

- 标记 `待核实`
- 给出可能来源
- 不要强行补全

## 推荐提示词

可以直接把下面这段交给 LLM：

```text
请根据收集/Inbox.md 和今天的日志：
1. 提取新的知识点，必要时写入知识目录；
2. 提取明确的个人感悟，必要时写入感悟目录；
3. 保留原始记录，不要删除；
4. 对不确定的知识标记为待核实；
5. 更新相关笔记的 related、tags、status；
6. 输出你做了哪些整理、哪些内容还需要我确认。
```
